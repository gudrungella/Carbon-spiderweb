# telecom_inventory

A Python toolset for managing and analysing telecom network inventory data. Supports Excel-based data entry with dropdown validation, schema validation via the telecom_api FastAPI server, emission factor enrichment from multiple providers, and CO2e analysis.

## Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
bash init_data.sh
```

`init_data.sh` copies the Excel templates from `templates/` into `data/`, which is where the pipeline reads and writes. This only needs to be run once after cloning.

## Data and templates

The `data/` folder is **gitignored** — nothing inside it is ever committed, no matter what. This means your inventory data stays local.

The `templates/` folder is tracked on GitHub. It contains the five input Excel files with column headers and sheet structure, but no data rows. These serve as the starting point for a fresh local setup.

| Folder | Tracked on GitHub | Purpose |
|---|---|---|
| `templates/` | Yes | Column structure and instructions — no data |
| `data/` | No | Your local inventory data — never committed |

If the schema changes (new columns added etc.), `templates/` is updated on GitHub. Pull the changes and re-run `init_data.sh` — it skips files that already exist, so your data is safe. To pick up the new columns in an existing file, copy the relevant template manually and re-enter your data.

## Workflow

### 1. Start telecom_api

The pipeline requires telecom_api to be running:

```bash
cd ../telecom_api
uvicorn main:app --host 0.0.0.0 --port 3000 --reload
```

### 2. Fill in data

Open the Excel files in `data/` and enter inventory data. Each file has:
- A **Data** sheet — fill in rows here (dropdowns guide valid values)
- An **Instructions** sheet — explains each column and valid values

### 3. Run the pipeline

```bash
python3 pipeline.py
```

This will:
1. Validate all four Excel files against the schema and POST valid rows to telecom_api
2. Print a row-by-row validation report with quality scores
3. Enrich missing emission factor and power fields from configured providers
4. Write enriched output files to `data/*_enriched.xlsx`
5. Append any unresolved fields to `emission_factors.xlsx` for manual entry

### 4. Manual enrichment fallback

If any fields could not be filled by the configured providers, `pipeline.py` appends placeholder rows to `data/emission_factors.xlsx`. Fill in the value for those rows and re-run — the custom file is always checked first.

### 5. Run CO2e analysis

```bash
python3 analyze.py
```

Loads the `*_enriched.xlsx` files and prints emission reports by component type, network type, and cell site. Writes results to `data/emissions_report.xlsx`.

## Running steps individually

```bash
# Validation only
python3 validate.py

# Convert Excel to CSV
python3 convert_excel.py
```

## Regenerating Excel input files

If the schema changes, regenerate all input Excel files with:

```bash
python3 create_excel.py
```

## API credentials (for emission factor and power enrichment)

Copy `.env.example` to `.env` and fill in the API keys you have:

```bash
cp .env.example .env
```

Emission factor providers (tried in order):

| Priority | Provider | Env var | Status |
|---|---|---|---|
| 1 | Custom file | `data/emission_factors.xlsx` — no key needed | Active |
| 2 | Rejoose | `REJOOSE_API_KEY` | Active |
| 3 | Climatiq | `CLIMATIQ_API_KEY` | Active |
| 4 | EPD | `EPD_API_KEY` | Stub — not yet implemented |
| 5 | Resilio | `RESILIO_API_KEY` | Stub — not yet implemented |
| 6 | Ecoinvent | `ECOINVENT_CLIENT_ID`, `ECOINVENT_CLIENT_SECRET` | Active |

Power field providers (`power_idle`, `power_max` on active components):

| Priority | Provider | Env var | Status |
|---|---|---|---|
| 1 | Custom file | `data/emission_factors.xlsx` — no key needed | Active |
| 2 | tscircuit | `TSCIRCUIT_API_KEY` | Stub — not yet implemented |

See `api_config.py` to reorder or disable providers.

### Country-specific emission factors

`emission_factors.xlsx` supports an optional `Country` column. When a country code is present in a row, that row is used only when enriching components from a matching country. Country-specific rows take priority over generic rows for the same component and field.

To add a country-specific factor, enter the ISO country code (e.g. `FR`, `DE`) in the `Country` column. Leave the column empty for a factor that applies to all countries.
